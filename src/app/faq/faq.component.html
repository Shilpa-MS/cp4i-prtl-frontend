General FAQs

Q1. What is JumpStart Kit?

TCS Integration Jumpstart Kit is a common marketplace for all the offerings and solutions under one hood.

Q2. Why do we need JumpStart?

The kit helps the customers to quickly adapt and simplify their integration modernization journey using Cloud Pak for Integration. TCS Integration architects and experts are continuously building reusable solutions and assets to address Technology Challenges and to find means to enable business adopt Modern Integration Solutions in a Quick and Risk Free Manner. The services and solutions listed here are the outcome of series of discussions and brain storming sessions with industry leaders and Architects to help the customers and account teams in modernization their integration journey.

Q3. IBM already have created Transformation Advisor for migration, what is the difference?

Jumpstart kit augment IBM TA by providing additional insights as below 1) Provides the target architecture model 2) Provides detailed migration roadmap 3) Provides ready to use tools and solution accelerators for workload migration into CP4I 4) Provides RoI

Q4. What is “One Click Deployment”?

One-click solution is designed for migrating existing IIB or ACE workload to IBM Cloudpak for Integration. It provides end-to-end automation that includes Checking out the Code from SCM system, Build deployable artifacts and Deploy to OpenShift container platform.

Q5. What is “Assessment Toolkit?

Assessment Toolkit is a starting point to assess customer(s) existing integration landscape. The tool breaks the entire integration ecosystem into technical, business, infrastructure and operations domains, analyzing the answers received and creates a comprehensive report detailing tools, accelerators, RoI and best practices, which can be leveraged towards customer integration modernization journey.

Q6. Is One-Click Deplpymet used for migration of existing integration projects or new development?

Existing and new integration workload can be migrated using One-click deployment. Developer has to configure appropriate source control repository inside One-click CI/CD process

Q7. How much saving we can achieve by using JumpStart?

Savings varies depending upon the kind of integration workload and number of message flows customer is running. Typically savings in the range of 10-30% should be realized as the journey progress

Q8. Can we JumpStart for all IBM Integration tools like IIB, MQ, Datapower and APIConnect?

Currently, jumostart kit has solutions and accelerators around IIB and MQ. It will be further enhanced to cover other products at a later stage

Q9. How can I access JumpStart Kit?

Jumpstart kit can be accessed after successful registration and verfication by Jumpstart admin. It can be accessed through https://cp4i-jumpstart-tcs.mybluemix.net/

Q10. What is refactoring?

Refactoring is a method to make changes in the old source code to make it work inside the current environment (e.g. IIB to ACE)

Q11. What is re-engineering?

Re-engineering is process where the existing source code is incompatible with the target environment and requires a major re-write/transformation

Q12. What is mean by Integration Pattern and what patterns are available?

Patterns are reusable flows which encapsulates common business logic and exposed as pattern. It can be integrated into any integration flow which has a similar business case. Patterns bring in synergy and standardization across integration flows (e.g. Error handling, MQ & Kafka pattern etc)

Q13. Can we get IBM Cloud Pak for Integration environment as part of JumpStat?

No. Jumpstart kit enables enterpries in faster adapting Cloud Pak for Integration to simplify and expedite their migration journey into CP4I.
